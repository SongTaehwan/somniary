#  Application
“비즈니스 규칙을 사용해서 앱이 실제로 무엇을 하게 할지”를 조립/실행하는 레이어.

> 앱이 비즈니스를 실행하는 방법(흐름/정책)

1. UseCase 실행/오케스트레이션
- 로그인, 가입, 프로필 수정 같은 흐름을 단계별로 수행

2. 정책
- 재시도/백오프, 트랜잭션 경계, 캐시 우선/네트워크 우선, 실패 분류/로깅 등

3. 에러
- UseCaseError
- Application Error

4. 유스케이스 관점의 매핑
- 도메인 에러 -> ContractError 축소
- Port Error -> UseCaseError 변환
- 유스케이스 결과 모델(Output)

5. 도메인 의존
- 도메인 모델과 포트를 사용해서 일을 한다

6. 외부 구현에 대한 의존은 없음
- 구체 네트워크/DB 구현체는 몰라야 함(그건 Data/Infra에서 주입)
