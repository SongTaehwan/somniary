## Common

### 목적

독립적으로 재사용 가능한 유틸리티 모듈의 집합

### 아키텍처 원칙

1. **모듈 독립성**: 각 하위 디렉토리는 독립된 모듈로 간주
2. **단방향 참조**: 명시적 의존성이 없는 한 다른 Common 모듈 참조 금지
3. **도메인 분리**: 프로젝트 특화 로직(Helpers) 참조 금지
4. **프레임워크 독립**: 외부 라이브러리 의존성 최소화

### 의존성 규칙

Common 모듈 → Core (아키텍처 인터페이스)

### 모듈 추가 기준

새로운 유틸리티 추가 시 다음을 확인:

- [ ] 다른 프로젝트에서도 사용 가능한가?
- [ ] 특정 비즈니스 로직에 종속되지 않는가?
- [ ] 독립적으로 테스트 가능한가?
- [ ] 단일 책임을 가지는가?

### 향후 계획

디렉토리 전체를 독립된 모듈로 분리 예정
